Create a database schema diagram and a clean folder structure for the “Phone Contact App” using Firebase (Firestore + Firebase Auth + Cloud Functions).

DATABASE: FIREBASE FIRESTORE SCHEMA

Design the schema as collections + documents. Use these definitions:

Collection: users

Each user document:

uid: string

email: string

createdAt: timestamp

Collection: contacts

Each document:

id: string

userId: string (owner)

name: string

phone: string

email: string

notes: string

createdAt: timestamp

updatedAt: timestamp

Index: userId

Collection: groups

Each document:

id: string

userId: string

name: string

description: string

backgroundInfo: string

contacts: array of contact IDs

schedule: array of objects:

date: timestamp

frequency: string (“weekly”, “monthly”, “yearly”, etc.)

createdAt: timestamp

updatedAt: timestamp

Index: userId

Collection: messageLogs

Each document:

id: string

userId: string

groupId: string

messageContent: string

recipients: array

timestamp: timestamp

Indexes: userId, groupId

FIREBASE CLOUD FUNCTIONS

Functions to scaffold:

generateAiMessage(groupId)

sendSms(groupId) (placeholder)

sendEmail(groupId) (placeholder)

scheduleMessageDispatch() using Cloud Scheduler

logMessageSend()

All functions must:

Validate user permissions

Use Firebase Admin SDK
